# Data Analyst Portfolio Plan: KYDEN KK

This document outlines the step-by-step plan to build a modern, high-impact data analyst portfolio for **KYDEN KK**, hosted on GitHub Pages.

## 1. Aesthetic & Branding
*   **Primary Theme**: Modern Turquoise (Default).
    *   *Accent Color*: `#00CED1` (Dark Turquoise).
    *   *Background*: Deep charcoal (`#121212`).
    *   *Icon*: Turquoise Turtle (🐢).
*   **Secondary Theme**: Light Pink (Toggleable).
    *   *Accent Color*: `#FFB6C1` (Light Pink).
    *   *Background*: Lavender Blush (`#FFF0F5`).
    *   *Icon*: Pink Bear (🐻).
*   **Typography**: 
    *   *Headings/Hero*: **Lobster Two** (via Google Fonts).
    *   *Body*: Clean sans-serif ('Inter') for professional readability.

## 2. Website Structure (Single Page)
### A. Hero Section
*   **Theme Toggle**: Fixed position (top right) switch to toggle between Turquoise and Pink themes.
*   **Content**: "KYDEN KK"
*   **Sub-headline**: "Data Analyst"
*   **Call to Action**: "View My Work" button leading to the Projects section.
*   **Styling**: Large, centered text using the Lobster Two font.

### B. Projects Section
*   **Layout**: A 3-column responsive grid.
*   **Content**: 3 Project Cards.
    *   *Card 1*: placeholder (e.g., Sales Performance Dashboard)
    *   *Card 2*: placeholder (e.g., Customer Segmentation Analysis)
    *   *Card 3*: placeholder (e.g., Financial Risk Modeling)
*   **Details**: Each card includes a title, short description, tool tags (Python, SQL, etc.), and a link to the GitHub repository.

### C. Contact Section
*   **Content**: Direct links to professional profiles.
    *   GitHub
    *   LinkedIn
    *   Email
*   **Footer**: Simple copyright notice.

## 3. Technical Stack
*   **HTML5**: Semantic structure.
*   **CSS3**: Custom styling with Flexbox/Grid. Uses CSS Variables for theme management.
*   **JavaScript**: Minimal vanilla JS for theme switching logic.

## 4. Implementation Steps
1.  **Initialize Repository**: Create a local folder and initialize it as a Git repository.
2.  **HTML Setup**: Create `index.html` with the base structure and link Google Fonts.
3.  **Styling**: Create `style.css` to implement the dual-theme system and responsive layout.
4.  **Theme Logic**: Implement `script.js` to handle the theme toggle functionality.
5.  **Content Population**: Add details for the 3 projects and contact info.
6.  **GitHub Upload**: Create a new repository on GitHub and push the code.
7.  **Deployment**: Enable GitHub Pages in the repository settings.
